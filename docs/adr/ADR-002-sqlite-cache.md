# ADR-002: キャッシュストレージに SQLite を採用する

- **日付**: 2026-06-19
- **状態**: Accepted

## 背景

探索済みカーネルのキャッシュには、KV ストア・ファイルシステム・SQLite・PostgreSQL 等が候補になる。
ローカル開発ツールとしての使いやすさと、将来の探索ログ分析が評価軸。

## 決定

**SQLite** (`~/.forge/cache.db`) をキャッシュストレージとして採用する。

## 理由

- 標準ライブラリのみで動作し、外部サービスが不要
- 単一ファイルなのでバックアップ・移動・削除が簡単
- SQL で探索ログを分析できる（「どのパラメータが何回 BEST になったか」等）
- 複数プロセスからの同時書き込みも WAL モードで対応可能
- KV ストア（pickle + ファイル）だと探索ログの集計が難しくなる

## トレードオフ

- 複数マシン間でキャッシュを共有する場合は rsync や別途同期が必要
- 非常に大きなカーネルコードを大量に保存する場合、ファイルサイズが増える（通常は問題ない）
